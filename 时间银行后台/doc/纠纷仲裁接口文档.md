# 纠纷仲裁接口文档

## 1. 概述
仲裁工作台允许管理员查看任务申诉详情，包括双方陈述、证据图片，并进行最终裁决（驳回、判给志愿者、退款给发布者）。

## 2. 数据库映射与分析

根据 `pages/AdjudicationWorkbench.tsx` 页面需求与 `doc/数据库现有表详情.md` 中的 `tb_appeal` 表，分析如下：

### 核心表: `tb_appeal` (申诉表)

| 前端字段 (Arbitration) | 数据库字段 (tb_appeal) | 说明 | 现状/建议 |
|---|---|---|---|
| id | id | 申诉ID | 现有 |
| taskId | task_id | 关联任务ID | 现有 |
| initiatorId | proposer_id | 发起人ID | 现有 |
| description | reason | 申诉详情/理由 | 现有 |
| evidenceImages | evidence_img | 证据图片 | **需变更**: 现有为 `varchar(255)` 单图，建议改为 `json` 或 `text` 存储多图 URL 数组。 |
| status | status | 状态 | 现有 (待处理/已通过/已驳回) |
| type | **无** | 申诉类型 (如: 拒不验收/虚假服务) | **缺字段**。建议新增 `type` 字段 (varchar)。 |
| defendantResponse | **无** | 被申诉人回应 | **缺字段**。建议新增 `defendant_response` 字段 (text)。 |
| createTime | create_time | 创建时间 | 现有 |

### 关联查询
- **任务信息**: 关联 `tb_task` 获取 `title`, `description`, `address`, `deadline`。
- **发起人信息**: 关联 `sys_user` 获取 `nickname`, `realName`, `role`。

## 3. 接口定义

### 3.1 获取仲裁详情

**URL**: `/arbitration/{id}`
**Method**: `GET`
**Description**: 获取单条仲裁案件的完整详情，包含关联的任务信息和证据。

**请求参数 (Path Params)**:

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | string | 是 | 仲裁单ID (tb_appeal.id) |

**响应数据**:

```typescript
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": "1",
    "taskId": "1001",
    "taskTitle": "陪老人去医院",
    "taskDescription": "周五上午去市医院...", // 任务快照
    "taskAddress": "北京市朝阳区...",
    "taskDeadline": "2024-05-21 18:00:00",
    "initiatorId": "U1001",
    "initiatorName": "张三",
    "initiatorRole": "VOLUNTEER", // 建议后端根据 proposer_id 在 task 中的角色判断返回
    "type": "拒不验收",
    "description": "我已经完成了服务...",
    "defendantResponse": "他根本没来...",
    "createTime": "2024-05-21 19:00:00",
    "status": "PENDING",
    "evidenceImages": [
        "http://xxx.com/img1.jpg",
        "http://xxx.com/img2.jpg"
    ]
  }
}
```

### 3.2 提交裁决

**URL**: `/arbitration/verdict`
**Method**: `POST`
**Description**: 管理员提交裁决结果。

**请求参数 (Body)**:

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | string | 是 | 仲裁单ID |
| verdictType | string | 是 | 裁决类型: `REJECT` (驳回/维持原判), `TO_VOLUNTEER` (判给志愿者/强制结算), `TO_PUBLISHER` (判给发布者/取消订单退款) |
| reason | string | 是 | 裁决理由/备注 |

**响应数据**:

```json
{
  "code": 200,
  "msg": "操作成功"
}
```

### 3.3 获取仲裁列表

**URL**: `/arbitration/list`
**Method**: `GET`
**Description**: 分页获取仲裁案件列表，支持状态筛选。

**请求参数 (Query Params)**:

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| page | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页条数，默认 10 |
| status | string | 否 | 状态筛选 (ALL, PENDING, RESOLVED) |
| keyword | string | 否 | 搜索关键词 (ID/任务标题/发起人) |

**响应数据**:

```typescript
{
  "code": 200,
  "msg": "success",
  "data": {
    "total": 50,
    "list": [
      {
        "id": "1",
        "taskId": "1001",
        "taskTitle": "陪老人去医院",
        "initiatorId": "U1001",
        "initiatorName": "张三",
        "initiatorRole": "VOLUNTEER",
        "type": "拒不验收",
        "createTime": "2024-05-21 19:00:00",
        "status": "PENDING"
      }
    ]
  }
}
```

## 1. 用户登录

**接口地址**: `/api/sys/login`  
**请求方式**: `POST`  
**描述**: 用户通过手机号和密码登录系统。

### 请求参数 (Body)

| 参数名 | 类型 | 必填 | 说明 |
|:---:|:---:|:---:|:---|
| phone | string | 是 | 11位手机号 |
| password | string | 是 | 密码 |

### 响应参数

| 参数名 | 类型 | 说明 |
|:---:|:---:|:---|
| code | int | 状态码 (200成功) |
| msg | string | 提示信息 |
| data | object | 响应数据 |
| └ token | string | 认证 Token |
| └ user | object | 用户信息 |
| &nbsp;&nbsp;└ id | string | 用户ID |
| &nbsp;&nbsp;└ phone | string | 手机号 |
| &nbsp;&nbsp;└ nickname | string | 昵称 |
| &nbsp;&nbsp;└ role | string | 用户角色 (VOLUNTEER, SENIOR, AGENT) |
| &nbsp;&nbsp;└ avatar | string | 头像URL |

### 示例

**请求**:
```json
{
  "phone": "13800138000",
  "password": "password123"
}
```

**响应 (成功)**:
```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR...",
    "user": {
      "id": "1001",
      "phone": "13800138000",
      "nickname": "张志愿者",
      "role": "VOLUNTEER",
      "avatar": "http://..."
    }
  }
}
```

## 2. 用户注册

**接口地址**: `/api/sys/register`  
**请求方式**: `POST`  
**描述**: 新用户注册，包含实名信息提交。

### 请求参数 (Body)

| 参数名 | 类型 | 必填 | 说明 |
|:---:|:---:|:---:|:---|
| phone | string | 是 | 手机号 |
| password | string | 是 | 密码 |
| nickname | string | 是 | 昵称 |
| realName | string | 是 | 真实姓名 |
| idCard | string | 是 | 身份证号 |
| idCardFront | string | 是 | 身份证**人像面**图片URL (需先调用 `/api/sys/upload` 上传，推荐调用 `/api/sys/ocr/idcard` 进行 OCR 识别回填信息) |
| idCardBack | string | 是 | 身份证**国徽面**图片URL (需先调用 `/api/sys/upload` 上传) |
| role | string | 是 | 角色: `SENIOR`, `VOLUNTEER`, `AGENT` (必须是大写英文) |

**注意**: 
1. 涉及图片上传字段 (`idCardFront`, `idCardBack`)，前端必须先调用 `/api/sys/upload` 获取 URL，再将 URL 传给注册接口。
2. 前端在获得身份证正反面 URL 后，可调用 `/api/sys/ocr/idcard` 接口识别并自动回填 `realName` 和 `idCard` 等字段。
3. 直接传输 Base64 数据会导致 500 错误 (数据过长)。

### 响应参数

| 参数名 | 类型 | 说明 |
|:---:|:---:|:---|
| code | int | 状态码 |
| msg | string | 提示信息 |
| data | object | 注册结果 |
| └ auditId | string | 审核ID |
| └ status | int | 初始审核状态 (0:待审核) |

### 示例

**请求**:
```json
{
  "phone": "13900139000",
  "password": "password123",
  "nickname": "李大爷",
  "realName": "李建国",
  "idCard": "110101195001011234",
  "idCardFront": "http://domain.com/uploads/front.jpg",
  "idCardBack": "http://domain.com/uploads/back.jpg",
  "role": "SENIOR"
}
```

**响应**:
```json
{
  "code": 200,
  "msg": "提交成功，请等待审核",
  "data": {
    "auditId": "10023",
    "status": 0
  }
}
```

## 3. 审核状态查询

**接口地址**: `/api/sys/audit/result`  
**请求方式**: `GET`  
**描述**: 轮询查询注册审核结果。

### 请求参数 (Query)

| 参数名 | 类型 | 必填 | 说明 |
|:---:|:---:|:---:|:---|
| auditId | string | 是 | 审核ID |

### 响应参数

| 参数名 | 类型 | 说明 |
|:---:|:---:|:---|
| code | int | 200 |
| data | object | |
| └ status | int | 审核状态 (0:待审核 1:已通过 2:已驳回) |
| └ reject_reason | string | 驳回原因 (当 status=2 时) |
| └ token | string | 登录Token (当 status=1 时) |
| └ user | object | 用户信息 (当 status=1 时) |

## 4. 身份证 OCR 识别 (用于自动填表)

**接口地址**: `/api/sys/ocr/idcard`  
**请求方式**: `POST`  
**描述**: 将获得的身份证访问URL传给后端进行阿里OCR识别，自动提取出人名、身份证号等内容，用于用户注册或亲情绑定环节的表单自动填充。

### 请求参数 (Body)

| 参数名 | 类型 | 必填 | 说明 |
|:---:|:---:|:---:|:---|
| imageUrl | string | 是 | 身份证图片访问URL |
| side | string | 是 | 人像面传 `face`，国徽面传 `back` |

### 响应参数

| 参数名 | 类型 | 说明 |
|:---:|:---:|:---|
| code | int | 200 |
| msg | string | 提示信息 |
| data | object | |
| └ name | string | 姓名 (仅 face 时返回) |
| └ idNum | string | 身份证号 (仅 face 时返回) |
| └ sex | string | 性别 |
| └ birth | string | 出生日期 |
| └ address | string | 地址 |
| └ issue | string | 签发机关 (仅 back 时返回) |
| └ startDate | string | 有效期起始日 (仅 back 时返回) |
| └ endDate | string | 有效期截止日 (仅 back 时返回) |
| └ side | string | front/back 原样返回 |

| file | file | 是 | 文件对象 |

### 响应参数

| 参数名 | 类型 | 说明 |
|:---:|:---:|:---|
| code | int | 200 |
| data | object | |
| └ url | string | 文件访问URL |

# 任务接单接口文档

## 1. 抢单/接取任务

**接口描述**: 志愿者接取处于"待接取"状态的任务。接取成功后，任务状态变更为"进行中"，并记录志愿者ID。

**请求方法**: `POST`
**请求路径**: `/api/task/accept`

**请求参数 (JSON)**:

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| taskId | string | 是 | 任务ID | 1735467890123 |
| userId | string | 是 | 志愿者ID (当前登录用户) | 2001 |

**响应参数 (JSON)**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | number | 状态码 (200成功, 500失败) |
| message | string | 响应信息 |
| data | boolean | 是否抢单成功 |

**响应示例**:
```json
{
  "code": 200,
  "message": "抢单成功",
  "data": true
}
```

**错误响应示例**:
```json
{
  "code": 500,
  "message": "手慢了，任务已被抢走",
  "data": false
}
```

## 2. 数据库变更说明
- **表**: `tb_task`
- **操作**: 
  - 更新 `volunteer_id` 为当前用户ID
  - 更新 `status` 从 `0` (待接取) 变为 `1` (进行中)
  - 条件: `id = taskId` AND `status = 0` (即乐观锁，确保任务未被他人接走)

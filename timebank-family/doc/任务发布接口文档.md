# 任务发布接口文档

## 1. 发布任务

**接口描述**: 用户发布新的服务需求任务

**请求方法**: `POST`
**请求路径**: `/api/task/publish`

**请求参数 (JSON)**:

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| title | string | 是 | 任务标题 | 帮我买袋米 |
| type | string | 是 | 服务类型 (CHAT, CLEANING, ERRAND, MEDICAL, OTHER) | ERRAND |
| description | string | 是 | 任务详细描述 | 需要购买一袋10kg的东北大米，送到302室 |
| coins | number | 是 | 悬赏积分 | 20 |
| location | string | 是 | 详细地址 (街道/小区/门牌号) | 幸福小区5号楼1单元302 |
| date | string | 是 | 服务日期 (YYYY-MM-DD) | 2024-12-30 |
| timeRange | string | 是 | 服务时间段 | 09:00 - 10:00 |
| publisherId | string | 否 | 发布者ID (用于指定发布人，如果不传则通过Token自动解析) | 1 |

**响应参数 (JSON)**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | number | 状态码 (200成功) |
| message | string | 响应信息 |
| data | object | 返回数据 |
| data.taskId | string | 新创建的任务ID |

**响应示例**:
```json
{
  "code": 200,
  "message": "发布成功",
  "data": {
    "taskId": "1735467890123"
  }
}
```

## 2. 获取用户余额

**接口描述**: 获取当前用户的最新积分余额

**请求方法**: `GET`
**请求路径**: `/api/user/balance`

**请求参数 (Query)**:

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| userId | string | 否 | 用户ID (如果不传则通过Token自动解析) | 1 |

**响应参数 (JSON)**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | number | 状态码 (200成功) |
| message | string | 响应信息 |
| data | object | 返回数据 |
| data.balance | number | 当前余额 |

**响应示例**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "balance": 150
  }
}
```

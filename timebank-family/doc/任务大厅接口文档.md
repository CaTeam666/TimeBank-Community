# 任务大厅接口文档

## 1. 获取任务大厅列表

**接口描述**: 获取任务大厅的任务列表，优先从 Redis 缓存获取，缓存没有则查询数据库

**请求方法**: `GET`
**请求路径**: `/api/task/hall`

**请求参数 (Query)**:

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| type | string | 否 | 任务类型过滤 (CHAT, CLEANING, ERRAND, MEDICAL, OTHER) - 前端需将中文类型映射为英文枚举 | ERRAND |

**响应参数 (JSON)**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | number | 状态码 (200成功) |
| message | string | 响应信息 |
| data | array | 任务列表 |
| data[].taskId | string | 任务ID |
| data[].title | string | 任务标题 |
| data[].type | string | 服务类型 |
| data[].description | string | 任务描述 |
| data[].coins | number | 悬赏积分 |
| data[].location | string | 服务地址 |
| data[].date | string | 服务日期 |
| data[].timeRange | string | 服务时间段 |
| data[].status | number | 任务状态 (0:待接取) |
| data[].publisherId | string | 发布者ID |
| data[].publisherName | string | 发布者昵称 |
| data[].createTime | string | 创建时间 |

**响应示例**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "taskId": "1735467890123",
      "title": "帮我买袋米",
      "type": "ERRAND",
      "description": "需要购买一袋10kg的东北大米，送到302室",
      "coins": 20,
      "location": "幸福小区5号楼1单元302",
      "date": "2024-12-30",
      "timeRange": "09:00 - 10:00",
      "status": 0,
      "publisherId": "1001",
      "publisherName": "张三",
      "createTime": "2024-12-29 10:30:00"
    }
  ]
}
```

## 2. 获取任务详情

**接口描述**: 根据任务ID获取任务详情

**请求方法**: `GET`
**请求路径**: `/api/task/detail`

**请求参数 (Query)**:

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| taskId | string | 是 | 任务ID | 1735467890123 |

**响应参数 (JSON)**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | number | 状态码 (200成功) |
| message | string | 响应信息 |
| data | object | 任务详情 |
| data.taskId | string | 任务ID |

## 3. 获取任务分类列表

**接口描述**: 获取任务大厅支持的所有任务分类

**请求方法**: `GET`
**请求路径**: `/api/task/categories`

**请求参数**: 无

**响应参数 (JSON)**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | number | 状态码 (200成功) |
| message | string | 响应信息 |
| data | array | 分类列表 |
| data[].key | string | 分类的英文枚举值 |
| data[].label | string | 分类的中文显示名称 |
| data[].icon | string | 分类图标名称（可选） |

**响应示例**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "key": "CHAT",
      "label": "陪聊",
      "icon": "message-circle"
    },
    {
      "key": "CLEANING",
      "label": "保洁",
      "icon": "zap"
    },
    {
      "key": "ERRAND",
      "label": "跑腿",
      "icon": "shopping-bag"
    },
    {
      "key": "MEDICAL",
      "label": "医疗陪护",
      "icon": "stethoscope"
    },
    {
      "key": "OTHER",
      "label": "其他",
      "icon": "briefcase"
    }
  ]
}
```

**备注**:
- 后端可以只返回具体的任务分类（CHAT、CLEANING等），不需要返回"全部"选项
- 前端会自动在列表首位添加"全部"选项，`key` 为 `ALL`
- 当用户点击"全部"时，前端调用任务列表接口时不传 `type` 参数
- `icon` 字段为前端图标建议，后端可选返回
- 前端应使用 `key` 字段调用其他API，使用 `label` 字段显示给用户

# 异常数据接口文档

## 1. 概述

本模块负责处理系统异常数据，目前主要包含僵尸任务（超时未接单被系统自动关闭）的管理与干预。

### 僵尸任务定义

- **异常任务**：服务开始时间前一小时还没有人接单的任务
- **僵尸任务**：到达服务时间仍未被接单的任务，系统自动取消并退款

### 自动检测机制

- 每30分钟自动检查Redis和数据库中的异常任务
- 服务时间前1小时内未接单的任务记录预警日志
- 到达服务时间未接单的任务自动取消并退款

---

## 2. 接口详情

### 2.1 获取僵尸任务日志列表

- **URL**: `/anomaly/zombie/logs`
- **Method**: `GET`
- **Description**: 获取被系统自动关闭的僵尸任务日志列表。

#### Response (Success)
```json
{
  "code": 200,
  "data": [
    {
      "id": "1",
      "taskId": "1001",
      "taskTitle": "陪聊任务",
      "closedTime": "2024-05-20 00:00:00",
      "refundAmount": 50,
      "refundStatus": "FAILURE"
    },
    {
      "id": "2",
      "taskId": "1002",
      "taskTitle": "保洁任务",
      "closedTime": "2024-05-21 00:00:00",
      "refundAmount": 100,
      "refundStatus": "SUCCESS"
    }
  ],
  "message": "success"
}
```

#### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 日志ID |
| taskId | String | 任务ID |
| taskTitle | String | 任务标题 |
| closedTime | String | 自动关闭时间 |
| refundAmount | Integer | 退款金额（时间币） |
| refundStatus | String | 退款状态（SUCCESS:成功, FAILURE:失败） |

---

### 2.2 手动触发僵尸任务检测

- **URL**: `/anomaly/zombie/check`
- **Method**: `POST`
- **Description**: 管理员手动触发僵尸任务检测，立即扫描并处理异常任务。

#### Request
无请求参数

#### Response (Success)
```json
{
  "code": 200,
  "data": null,
  "message": "success"
}
```

---

### 2.3 手动重试退款

- **URL**: `/anomaly/zombie/retry/{logId}`
- **Method**: `POST`
- **Description**: 对退款失败的僵尸任务进行手动重试退款。

#### 路径参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| logId | Long | 是 | 僵尸任务日志ID |

#### Response (Success)
```json
{
  "code": 200,
  "data": true,
  "message": "success"
}
```

#### Response (Failure)
```json
{
  "code": 200,
  "data": false,
  "message": "success"
}
```

---

## 3. 状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 500 | 服务器内部错误 |

---

## 4. 退款状态说明

| 状态 | 说明 |
|------|------|
| SUCCESS | 退款成功，时间币已返还给发布者 |
| FAILURE | 退款失败，需要管理员手动重试 |

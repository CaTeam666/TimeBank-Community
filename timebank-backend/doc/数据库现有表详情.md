## 表关系说明

### 核心业务流程

1. **用户关系管理**
   - `sys_user`: 存储所有用户信息（老人、志愿者、子女代理人）
   - `sys_user_relation`: 关联子女代理人与老人的关系

2. **任务流程**
   - 老人在 `tb_task` 发布任务
   - 志愿者接取任务
   - 任务完成后更新状态
   - 通过 `tb_coin_log` 记录时间币流水

3. **申诉流程**
   - 任务产生争议时，在 `tb_appeal` 创建申诉记录
   - 关联 `tb_task` 和发起人

4. **兑换流程**
   - 用户在 `tb_product` 选择商品
   - 创建 `tb_exchange_order` 订单
   - 通过 `tb_coin_log` 扣减时间币

### 表关联关系

```
sys_user (用户表)
  ├── sys_user_relation.child_id (子女代理人关系)
  ├── sys_user_relation.parent_id (父母老人关系)
  ├── tb_task.publisher_id (发布的任务)
  ├── tb_task.volunteer_id (接取的任务)
  ├── tb_appeal.proposer_id (发起的申诉)
  ├── tb_coin_log.user_id (时间币流水)
  └── tb_exchange_order.user_id (兑换订单)

tb_task (任务表)
  ├── tb_appeal.task_id (任务申诉)
  └── tb_coin_log.task_id (任务相关流水)

tb_product (商品表)
  └── tb_exchange_order.product_id (兑换订单)
```

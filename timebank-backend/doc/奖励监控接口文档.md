# 奖励监控接口文档

## 1. 接口概述

Base URL: `/api`

本模块负责查询每月自动发放的奖励记录及异常重试。

## 2. 数据结构

### RankingLog (排名日志)

```typescript
interface RankingLog {
  id: string;
  period: string;        // 期数 (e.g., "2024-11")
  rank: number;          // 排名
  volunteerId: string;
  volunteerName: string;
  volunteerAvatar: string;
  orderCount: number;    // 接单数 (原服务时长)
  rewardAmount: number;  // 奖励金额
  distributionTime: string; // 发放时间
  status: 'SUCCESS' | 'FAILURE';
}
```

## 3. 接口详情

### 3.1 获取奖励发放日志

优先从Redis缓存查询，若缓存不存在则查询数据库。

- **URL**: `/ranking/logs`
- **Method**: `GET`
- **Query Params**:
  - `period`: 期数筛选 (可选，格式: YYYY-MM)
  - `page`: 页码
  - `pageSize`: 每页数量

> **缓存说明**: 当指定`period`参数时，系统会优先从Redis缓存(key: `ranking:monthly:{period}`)获取数据，缓存有效期90天。

#### Response
```json
{
  "code": 200,
  "data": {
      "list": [
          {
              "id": "1",
              "period": "2024-11",
              "rank": 1,
              "volunteerName": "李四",
              "orderCount": 45,
              "rewardAmount": 100,
              "status": "SUCCESS",
              "distributionTime": "2024-12-01 00:00:05"
          }
      ],
      "total": 5
  },
  "message": "success"
}
```

### 3.2 手动触发补发

- **URL**: `/ranking/retry`
- **Method**: `POST`
- **Content-Type**: `application/json`

#### Request Body
```json
{
  "id": "1" // 日志ID
}
```

#### Response
```json
{
  "code": 200,
  "message": "补发指令已提交"
}
```

### 3.3 手动触发月度排名统计

用于补跑指定月份的排名数据，将统计上月接单数前5名的志愿者并发放时间币奖励。

- **URL**: `/ranking/statistics`
- **Method**: `POST`
- **Query Params**:
  - `period`: 期数 (格式: YYYY-MM，如 2026-01)

#### Response
```json
{
  "code": 200,
  "message": "月度排名统计完成"
}
```

#### 奖励规则

| 排名 | 奖励时间币 |
|------|------------|
| 第1名 | 1000 |
| 第2名 | 500 |
| 第3名 | 300 |
| 第4名 | 300 |
| 第5名 | 300 |

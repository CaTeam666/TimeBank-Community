## 1. 认证模块

### 1.1 用户登录
- **URL**: `/auth/login`
- **Method**: `POST`
- **Description**: 用户登录接口。
- **Request Body**:
  - `username` (string): 用户名
  - `password` (string): 密码
- **Response**:
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "token": "...",
    "user": {
      "id": "...",
      "username": "...",
      "role": "..."
    }
  }
}
```

## 2. 用户管理

### 2.1 获取用户列表
- **URL**: `/users`
- **Method**: `GET`
- **Description**: 分页获取用户列表，支持筛选和搜索。
- **Params**:
  - `page` (number): 页码 (默认1)
  - `pageSize` (number): 页大小 (默认10)
  - `keyword` (string): 搜索关键字 (昵称/手机/姓名)
  - `role` (string): 角色筛选 (ELDER, VOLUNTEER, CHILD_AGENT, ADMIN)
  - `status` (string): 状态筛选 (NORMAL, FROZEN)
- **Response**:
```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": "u1",
        "avatar": "https://example.com/avatar1.jpg",
        "nickname": "乐于助人",
        "realName": "张三",
        "phone": "13800138000",
        "role": "VOLUNTEER",
        "balance": 150,
        "registerTime": "2023-11-20 10:00:00",
        "status": "NORMAL"
      }
    ],
    "total": 100
  }
}
```

### 2.2 获取用户详情
- **URL**: `/users/{id}`
- **Method**: `GET`
- **Params**: `id` (path)
- **Response**: 用户完整信息对象。

### 2.3 更新用户状态 (冻结/解冻)
- **URL**: `/users/{id}/status`
- **Method**: `PATCH`
- **Body**: `{ "status": "FROZEN" }` (NORMAL/FROZEN)

### 2.4 调整用户余额
- **URL**: `/users/{id}/balance-adjustment`
- **Method**: `POST`
- **Body**: `{ "amount": 50, "reason": "奖励" }` (amount可正可负)

## 3. 实名审核

### 3.1 获取实名审核列表
- **URL**: `/identity-audits`
- **Method**: `GET`
- **Params**:
  - `page`, `pageSize`
  - `status` (PENDING, APPROVED, REJECTED)
- **Response**: `{ "list": [...], "total": ... }`

### 3.2 获取审核详情
- **URL**: `/identity-audits/{id}`
- **Method**: `GET`
- **Response**: 审核任务详情，包含OCR信息和身份证图片。

### 3.3 提交审核结果
- **URL**: `/identity-audits/{id}/audit`
- **Method**: `POST`
- **Body**:
  - `status` (string): APPROVED / REJECTED
  - `rejectReason` (string): 拒绝原因 (当 REJECTED 时必填)

## 4. 亲情绑定

### 4.1 获取亲情绑定申请列表
- **URL**: `/user-relations`
- **Method**: `GET`
- **Params**:
  - `page`, `pageSize`
  - `status` (0:待审核, 1:已通过, 2:已拒绝)
  - `keyword`
- **Response**: `{ "list": [...], "total": ... }`

### 4.2 审核亲情绑定申请
- **URL**: `/user-relations/{id}/audit`
- **Method**: `POST`
- **Body**:
  - `status` (number): 1 (通过) / 2 (拒绝)
  - `rejectReason` (string): 拒绝原因

## 5. 任务管理 (任务监控)

### 5.1 获取任务列表
- **URL**: `/mission/list`
- **Method**: `GET`
- **Params**:
  - `page`, `pageSize`
  - `keyword`, `status`, `type`, `date`
- **Response**: 包含发布者、志愿者信息的任务列表。

### 5.2 获取任务详情
- **URL**: `/mission/{id}`
- **Method**: `GET`
- **Response**: 任务详情，包含日志 (logs)。

### 5.3 强制关闭任务
- **URL**: `/mission/force-close`
- **Method**: `POST`
- **Body**: `{ "taskId": "...", "reason": "..." }`

## 7. 纠纷仲裁

### 7.1 获取仲裁列表
- **URL**: `/arbitration/list`
- **Method**: `GET`
- **Params**: `page`, `pageSize`, `status`, `keyword`
- **Response**: 申诉案件列表。

### 7.2 获取仲裁详情
- **URL**: `/arbitration/{id}`
- **Method**: `GET`
- **Response**: 包含任务快照、双方陈述、证据图片的完整案件详情。

### 7.3 提交裁决
- **URL**: `/arbitration/verdict`
- **Method**: `POST`
- **Body**:
  - `id`: 仲裁单ID
  - `verdictType`: REJECT (驳回/维持原判), TO_VOLUNTEER (判给志愿者), TO_PUBLISHER (判给发布者)
  - `reason`: 裁决理由

## 8. 商品管理

### 8.1 获取商品列表
- **URL**: `/product/list`
- **Method**: `GET`
- **Response**: 包含所有商品（含下架）。

### 8.2 新增商品
- **URL**: `/product/create`
- **Method**: `POST`
- **Body**: `{ "name": "...", "price": 100, "stock": 10, "image": "...", "description": "..." }`

### 8.3 更新商品
- **URL**: `/product/update`
- **Method**: `POST`
- **Body**: `{ "id": "...", "name": "..." }` (仅发送需修改字段)

### 8.4 上下架商品
- **URL**: `/product/status`
- **Method**: `POST`
- **Body**: `{ "id": "...", "status": "ON_SHELF" | "OFF_SHELF" }`

### 8.5 删除商品
- **URL**: `/product/delete`
- **Method**: `POST`
- **Body**: `{ "id": "..." }`

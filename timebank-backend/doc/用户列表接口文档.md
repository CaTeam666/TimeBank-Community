# 用户列表接口文档

## 1. 概述
本文档描述了时间银行后台管理系统中用户列表模块的相关后端接口。

**Base URL**: `/api`

## 2. 数据模型

### 2.1 User (用户)

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | string | 用户唯一标识 |
| `avatar` | string | 头像URL |
| `nickname` | string | 昵称 |
| `realName` | string | 真实姓名 |
| `phone` | string | 手机号 |
| `role` | enum | 用户角色 (见 2.2) |
| `balance` | number | 时间币余额 |
| `registerTime` | string | 注册时间 (ISO 8601或标准时间格式) |
| `status` | enum | 账号状态 (见 2.3) |

### 2.2 UserRole (用户角色枚举)
- `ELDER`: 老人
- `VOLUNTEER`: 志愿者
- `CHILD_AGENT`: 子女代理人
- `ADMIN`: 管理员

### 2.3 AccountStatus (账号状态枚举)
- `NORMAL`: 正常
- `FROZEN`: 冻结

---

## 3. 接口详情

### 3.1 获取用户列表

支持分页、关键词搜索和筛选。

- **URL**: `/users`
- **Method**: `GET`

**请求参数 (Query Parameters)**:

| 参数名 | 类型 | 必选 | 说明 |
| :--- | :--- | :--- | :--- |
| `page` | number | 否 | 页码，默认 1 |
| `pageSize` | number | 否 | 每页数量，默认 10 |
| `keyword` | string | 否 | 搜索关键词 (匹配昵称、手机号或真实姓名) |
| `role` | string | 否 | 角色筛选 (`ELDER`, `VOLUNTEER`, `CHILD_AGENT`, `ADMIN`) |
| `status` | string | 否 | 状态筛选 (`NORMAL`, `FROZEN`) |

**响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "id": "u1",
        "avatar": "https://example.com/avatar1.jpg",
        "nickname": "乐于助人",
        "realName": "张三",
        "phone": "13800138000",
        "role": "VOLUNTEER",
        "balance": 150,
        "registerTime": "2023-11-20 10:00:00",
        "status": "NORMAL"
      }
    ],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

### 3.2 获取用户详情

- **URL**: `/users/{id}`
- **Method**: `GET`

**路径参数**:
- `id`: 用户ID

**响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "u1",
    "avatar": "https://example.com/avatar1.jpg",
    "nickname": "乐于助人",
    "realName": "张三",
    "phone": "13800138000",
    "role": "VOLUNTEER",
    "balance": 150,
    "registerTime": "2023-11-20 10:00:00",
    "status": "NORMAL"
  }
}
```

### 3.3 更新用户状态 (冻结/解冻)

- **URL**: `/users/{id}/status`
- **Method**: `PATCH`

**请求体 (JSON)**:

| 字段名 | 类型 | 必选 | 说明 |
| :--- | :--- | :--- | :--- |
| `status` | string | 是 | 目标状态 (`NORMAL` 或 `FROZEN`) |

**示例 payload**:
```json
{
  "status": "FROZEN"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "状态更新成功"
}
```

### 3.4 调整用户余额

管理员对用户余额进行增加或扣除。

- **URL**: `/users/{id}/balance-adjustment`
- **Method**: `POST`

**请求体 (JSON)**:

| 字段名 | 类型 | 必选 | 说明 |
| :--- | :--- | :--- | :--- |
| `amount` | number | 是 | 调整金额。正数为增加，负数为扣除。 |
| `reason` | string | 否 | 调整原因备注 |

**示例 payload**:
```json
{
  "amount": 50,
  "reason": "参与活动奖励"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "余额调整成功",
  "data": {
    "currentBalance": 200
  }
}
```

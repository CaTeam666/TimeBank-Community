# API 接口文档

## 认证模块

### 用户登录

**接口地址**: `/auth/login`
**请求方式**: `POST`

**请求参数** (Request Body):

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| username | string | 是 | 用户名 (如: admin) |
| password | string | 是 | 密码 (如: 123456) |

**响应参数** (Response Body):

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| code | number | 状态码 (200表示成功) |
| message | string | 提示信息 |
| data | object | 数据载体 |
| data.token | string | 身份令牌 |
| data.user | object | 用户信息对象 |

**登录成功示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "admin-jwt-token-ADMIN001-1740638400000",
    "user": {
      "id": "ADMIN001",
      "realName": "管理员",
      "nickname": "admin",
      "role": "ADMIN",
      "phone": "13800000000",
      "email": "admin@example.com",
      "avatar": null
    }
  },
  "timestamp": 1740638400000
}
```

**登录错误响应**:

| code | message | 说明 |
|------|---------|------|
| 401 | 账号或密码错误 | 用户名或密码不正确 |
| 403 | 用户已被禁用 | 用户账户已被停用 |

## 令牌验证说明

### Token 格式

系统端 Token 格式为: `admin-jwt-token-{userId}-{timestamp}`

### 请求头要求

所有需要认证的接口必须在请求头中携带 Token：

```
Authorization: admin-jwt-token-ADMIN001-1740638400000
```

### 认证失败响应（前端需识别并跳转登录页）

以下响应表示用户需要重新登录，前端应识别 `code: 401` 并跳转到登录页：

| code | message | 说明 |
|------|---------|------|
| 401 | 请先登录 | 请求未携带 Authorization header |
| 401 | 无效的令牌 | Token 格式不正确或非系统端 Token |
| 401 | 令牌已过期或无效，请重新登录 | Token 已过期或在其他设备登录 |

**认证失败响应示例**:

```json
{
  "code": 401,
  "message": "令牌已过期或无效，请重新登录",
  "data": null,
  "timestamp": 1740638400000
}
```

### 前端处理建议

```typescript
// 在响应拦截器中处理 401 错误
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401 || error.response?.data?.code === 401) {
      // 清除本地存储的 token
      localStorage.removeItem('token');
      // 跳转到登录页
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
```

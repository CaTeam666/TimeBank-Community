# 数据驾驶舱接口文档

## 基础信息
- **接口前缀**: `/api/admin/dashboard`
- **鉴权要求**: 需要系统端管理员 Token

---

## 1. 获取核心指标统计 (KPI卡片)

获取数据驾驶舱顶部的四个核心统计指标数据。

- **接口地址**: `/kpi`
- **请求方法**: `GET`
- **请求参数**: 无

### 响应参数
| 参数名称 | 类型 | 说明 |
|----------|------|------|
| code | Integer | 状态码，200为成功 |
| message | String | 提示信息 |
| data | Object | 核心指标数据 |
| └─ totalPopulation | Long | 社区总人口（系统当前已注册的用户总数） |
| └─ todayExchangeCount | Integer | 爱心超市兑换数（今日已经兑换的实物数量） |
| └─ todayNewOrders | Integer | 今日新增订单（当日实时完成的订单数量） |
| └─ todayPointsCirculation | Integer | 今日积分流通（今日实时的积分流水总额） |

#### 响应示例
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "totalPopulation": 3580,
    "todayExchangeCount": 124,
    "todayNewOrders": 45,
    "todayPointsCirculation": 5800
  }
}
```

## 2. 获取近7日服务活跃度趋势

查询过去七天内每天的订单完成数量，用于绘制折线图。

- **接口地址**: `/trend/activity`
- **请求方法**: `GET`
- **请求参数**: 无

### 响应参数
| 参数名称 | 类型 | 说明 |
|----------|------|------|
| code | Integer | 状态码，200为成功 |
| message | String | 提示信息 |
| data | Array | 每日统计列表数据，按日期正序排列 |
| └─ date | String | 统计日期，格式：yyyy-MM-dd |
| └─ count | Integer | 当日完成的订单数 |

#### 响应示例
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "date": "2026-03-16",
      "count": 35
    },
    {
      "date": "2026-03-17",
      "count": 42
    }
  ]
}
```

## 3. 获取任务类型分布

获取系统中各类型任务的分布比例和数量，用于绘制饼图。（数据结构暂不变更）

- **接口地址**: `/distribution/task-type`
- **请求方法**: `GET`
- **请求参数**: 无

### 响应参数
| 参数名称 | 类型 | 说明 |
|----------|------|------|
| code | Integer | 状态码，200为成功 |
| message | String | 提示信息 |
| data | Array | 类型分布数据列表 |
| └─ typeName | String | 任务类型名称（如：生活照料、精神慰藉等） |
| └─ count | Integer | 该类型任务的总数 |

#### 响应示例
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "typeName": "生活照料",
      "count": 400
    },
    {
      "typeName": "精神慰藉",
      "count": 250
    }
  ]
}
```

## 4. 获取实时动态

查询系统实时收到的重要申请和事件处理（如：注册申请、亲情绑定申请、服务仲裁纠纷等），用于首页滚动播报。

- **接口地址**: `/dynamics`
- **请求方法**: `GET`
- **请求参数**: 
| 参数名称 | 必填 | 类型 | 说明 |
|----------|------|------|------|
| limit | 否 | Integer | 返回的动态条数，默认返回最新的20条 |

### 响应参数
| 参数名称 | 类型 | 说明 |
|----------|------|------|
| code | Integer | 状态码，200为成功 |
| message | String | 提示信息 |
| data | Array | 实时动态列表数据，按时间倒序排列 |
| └─ id | Long | 动态记录的唯一ID |
| └─ type | Integer | 动态类型：1-注册申请，2-亲情绑定申请，3-服务仲裁纠纷 |
| └─ userName | String | 发起人或相关人姓名 |
| └─ content | String | 动态业务内容描述 |
| └─ createTime| String | 发生时间，格式：yyyy-MM-dd HH:mm:ss |

#### 响应示例
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 10001,
      "type": 1,
      "userName": "张阿姨",
      "content": "提交了志愿者注册申请",
      "createTime": "2026-03-22 10:05:00"
    },
    {
      "id": 10002,
      "type": 3,
      "userName": "李大爷",
      "content": "发起了服务仲裁纠纷",
      "createTime": "2026-03-22 09:30:00"
    }
  ]
}
```

# 时间银行后端项目结构剖析

**项目名称**: ai-end  
**技术栈**: Spring Boot 3.1.5 + MyBatis-Plus 3.5.5 + MySQL + Redis  
**JDK版本**: 17  
**更新时间**: 2026-03-14

---

## 一、项目定位

**时间银行互助养老平台后端服务**，提供老人发布服务需求、志愿者接单服务、子女代理管理、时间币流转、爱心超市兑换等核心功能的 RESTful API。

## 四、核心业务模块

### 4.1 用户体系

| 角色 | 说明 | 主要功能 |
|------|------|----------|
| 老人 | 服务需求方 | 发布任务、验收服务、兑换商品 |
| 志愿者 | 服务提供方 | 接取任务、提交凭证、获取时间币 |
| 子女代理人 | 代理操作 | 绑定老人、代理发布/管理任务 |
| 管理员 | 后台管理 | 审核、监控、仲裁、配置 |

### 4.2 任务流程

```
发布任务 → 待接单(0) → 志愿者接单 → 进行中(1) → 提交凭证 → 待验收(2)
                                                        │
                    ┌───────────────────────────────────┴───────────────────┐
                    ▼                                                       ▼
               老人验收通过                                             老人拒绝/申诉
                    │                                                       │
                    ▼                                                       ▼
               已完成(3)                                                申诉中(4)
           (时间币转给志愿者)                                         (管理员仲裁)
```

### 4.3 亲情绑定流程

```
子女申请 → 待管理员审核(0) → 管理员通过 → 待老人确认(1) → 老人同意 → 已绑定(2)
                │                              │
                ▼                              ▼
            管理员拒绝                       老人拒绝
                │                              │
                └──────────── 已拒绝(3) ────────┘
```

### 4.4 时间币体系

| 流水类型 | 说明 |
|----------|------|
| 1-任务收入 | 志愿者完成任务获得 |
| 2-任务支出 | 老人发布任务冻结/支付 |
| 3-兑换支出 | 爱心超市兑换消费 |
| 4-系统调整 | 老人实名通过赠送、排名奖励等 |

## 五、认证机制

### 5.1 系统端认证

- **Token格式**: `admin-jwt-token-{userId}-{timestamp}`
- **存储**: Redis (24小时过期)
- **拦截器**: `AdminAuthInterceptor`
- **请求头**: `Authorization: {token}`

### 5.2 用户端认证

- **Token格式**: `client-jwt-token-{userId}-{timestamp}`
- **存储**: Redis (7天过期)
- **代理模式**: 支持 `X-Real-User-Id` 头实现子女代理老人操作

## 六、定时任务

| 调度器 | 执行周期 | 功能 |
|--------|----------|------|
| RealTimeRankingScheduler | 每5分钟 | 更新实时接单排行榜 (Redis) |
| MonthlyRankingScheduler | 每月1日0点 | 结算上月排名并发放奖励 |
| ZombieTaskScheduler | 每小时 | 关闭超时未接单任务，自动退款 |
| TaskDelayDeleteScheduler | 每天凌晨 | 清理已取消超过7天的任务 |
| CoinFlowStatisticsScheduler | 每5分钟 | 统计时间币流通数据 (Redis) |

## 七、Redis 缓存设计

| Key | 类型 | 说明 |
|-----|------|------|
| `admin:token:{userId}` | String | 系统端登录Token |
| `client:token:{userId}` | String | 用户端登录Token |
| `ranking:realtime:{yyyy-MM}` | ZSet | 当月实时接单排行榜 |
| `stats:coin:daily` | Hash | 今日时间币统计 |

## 十、快速启动

```bash
# 1. 配置数据库连接 (application-dev.yml)
spring.datasource.url=jdbc:mysql://localhost:3306/ai_end
spring.datasource.username=root
spring.datasource.password=your_password

# 2. 配置Redis
spring.redis.host=localhost
spring.redis.port=6379

# 3. 启动项目
mvn spring-boot:run -Dspring-boot.run.profiles=dev

# 4. 访问测试
curl http://localhost:8080/hello
```

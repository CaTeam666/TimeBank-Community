# 后端代理模式任务清单

> 本文档面向后端开发，描述前端代理模式改造后，后端需要配合完成的工作。

---

## 背景

前端已完成代理模式的身份切换：代理人操作时，请求头携带 `proxyToken`（JWT），其中 `userId` 为被代理老人 ID，`realUserId` 为实际操作的子女 ID。

前端在代理模式下的所有白名单请求均已使用老人身份（userId/publisherId 均传老人ID），并发送 `proxyToken` 作为 `Authorization: Bearer {proxyToken}`。

---

## 任务一：ProxyToken 解析与身份识别

### 目标
白名单接口中，后端应从 `proxyToken` 的 JWT payload 中提取 `userId` 作为当前操作用户，而非依赖请求 body/params 中的 userId。

### 具体要求

1. **Token 解析逻辑**：
   - 检查 JWT 中是否存在 `isProxy = true`
   - 若是代理 Token，使用 `userId`（老人ID）作为操作身份
   - 记录 `realUserId`（子女ID）到审计日志

2. **受影响接口（白名单）**：

| 接口路径 | 方法 | 说明 |
|---------|------|------|
| `/api/task/publish` | POST | 发布任务 — `publisherId` 应以 Token 中 userId 为准 |
| `/api/task/my/published` | GET | 查看老人发布的任务 |
| `/api/task/my/accepted` | GET | 查看老人接的任务 |
| `/api/user/balance` | GET | 查询老人的积分余额 |
| `/api/mall/products` | GET | 查看商品列表 |
| `/api/mall/exchange` | POST | 用老人积分兑换商品 |
| `/api/mall/orders` | GET | 查看老人的兑换订单 |

3. **非白名单接口**：收到 `proxyToken` 时应返回 `403 Forbidden`

---

## 任务二：代理发布任务限流

### 目标
限制每个代理人每日对同一老人的代理发布任务数量，防止滥用。

### 具体要求

1. **新增配置项**：`proxy.daily.publish.limit`，建议默认值为 **3**（每个代理人每天最多为同一老人发布3个任务）

2. **限流维度**：`(realUserId, userId, date)` —— 即某个子女对某个老人在某天

3. **实现方式建议**：
   - 在 `task/publish` 接口中，如果检测到 `isProxy = true`：
     - 查询当天该 `realUserId` 为该 `userId` 已发布的任务数
     - 若超过上限，返回 `429 Too Many Requests` 或 `403` 并附带提示信息
   
4. **错误响应格式**：
```json
{
  "code": 429,
  "message": "您今日代理发布任务已达上限（3次），请明日再试",
  "data": null
}
```

---

## 任务三：审计日志

### 目标
所有代理操作均应留下完整的审计痕迹。

### 具体字段

| 字段 | 类型 | 说明 |
|------|------|------|
| userId | bigint | 被代理人ID（老人） |
| realUserId | bigint | 实际操作人ID（子女） |
| action | varchar | 操作类型（如 PUBLISH_TASK, EXCHANGE） |
| detail | text | 操作详情（如任务ID、商品ID等） |
| createTime | datetime | 操作时间 |
| ip | varchar | 请求来源IP |

### 建议

- 可新建 `sys_proxy_audit_log` 表
- 在所有白名单接口的代理分支中插入审计记录
- 用于后续争议追溯和安全审查

---

## 任务四：parentId 类型兼容

### 背景
前端已将 `/api/family/proxy/toggle` 接口中 `parentId` 从 `number` 改为 `string` 传递（因 JS 大整数精度丢失问题）。

### 后端调整
- `ProxyToggleDTO` 中的 `parentId` 字段改为 `String` 类型接收
- 后端内部转换为 `Long` 使用

---

## 优先级

| 优先级 | 任务 | 原因 |
|-------|------|------|
| P0 | 任务一（ProxyToken 解析） | 前端已改完，后端不配合则代理功能不可用 |
| P0 | 任务四（parentId 类型） | 大整数精度问题导致代理功能完全不可用 |
| P1 | 任务二（限流） | 防止滥用，上线前必须有 |
| P2 | 任务三（审计日志） | 运营和安全需求，可稍后补充 |

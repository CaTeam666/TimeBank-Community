# 消息中心接口文档

## 概述

消息中心模块提供统一的消息管理功能，用于展示用户的待办消息。消息会在业务处理完成后自动删除。

**消息类型说明**:
| 类型码 | 类型名称 | 说明 | 跳转路由 |
|--------|----------|------|----------|
| FAMILY_BIND | 亲情绑定 | 收到亲情账号绑定申请 | /pages/family/pending-requests |
| TASK_VERIFY | 任务验收 | 任务待验收通知 | /pages/task/verify |

**消息生命周期**:
- 亲情绑定消息：管理员审核通过后创建，被绑定方确认/拒绝后自动删除
- 任务验收消息：志愿者提交凭证后创建，发布者确认验收后自动删除

## 1. 获取个人消息列表

**接口路径**: `GET /api/message/list`

**功能说明**: 获取当前用户的待办消息列表，支持按消息类型筛选，优先从Redis缓存获取

**请求头**:
```
Authorization: Bearer {token}
```

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| type | string | 否 | 消息类型（FAMILY_BIND/TASK_VERIFY），不传则获取全部类型 |
| pageNum | integer | 否 | 页码，默认1 |
| pageSize | integer | 否 | 每页条数，默认10，最大50 |

**响应参数**:
| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | integer | 状态码 |
| message | string | 响应消息 |
| data | object | 响应数据 |
| data.total | integer | 总记录数 |
| data.list | array | 消息列表 |
| data.list[].id | long | 消息ID |
| data.list[].type | string | 消息类型 |
| data.list[].typeName | string | 消息类型名称 |
| data.list[].bizId | long | 关联业务ID |
| data.list[].title | string | 消息标题 |
| data.list[].content | string | 消息内容 |
| data.list[].route | string | 跳转路由 |
| data.list[].createTime | string | 创建时间 |

**响应示例**:

## 缓存策略说明

1. **Redis缓存Key设计**:
   - 用户全部消息：`message:user:{userId}:all`
   - 用户按类型消息：`message:user:{userId}:type:{type}`

2. **缓存更新策略**:
   - 创建消息时：同步写入Redis
   - 删除消息时：清除相关缓存
   - 缓存过期时间：24小时

3. **查询优先级**:
   - 优先查询Redis缓存
   - 缓存未命中时查询数据库并回写缓存

# 系统核心功能实现逻辑文档

本文档详细说明了“互助养老”系统针对老人角色的积分奖励机制及其底层技术实现逻辑。

---

## 1. 老人注册初始积分赠送逻辑

老人的注册奖励并非在“提交表单”时立即发放，而是与**实名身份核验**流程强绑定，确保资产发放的安全真实性。

### 1.1 触发流程
1.  **提交注册**：用户在端侧提交实名信息及证件照片（`ClientAuthServiceImpl.register`），此时 `User.balance` 初始为 `0`，账号状态为禁用。
2.  **后台审核**：管理员在后台管理端核实证件信息后，点击“审核通过”。
3.  **自动赠送**：系统在审核状态变更为 `APPROVED` 的事务中，自动触发积分划转。

### 1.2 核心逻辑实现
*   **代码位置**：`IdentityAuditServiceImpl.grantElderInitialCoinsIfElder(Long userId)`
*   **角色判定**：严格检查 `sys_user.role` 是否包含“老人”关键字，其他角色（如志愿者）不享受此项福利。
*   **配置获取**：通过 `SettingsService` 获取后台动态配置参数 `elder_initial_coins`（系统默认值：**100**）。
*   **原子更新**：使用 MyBatis-Plus 的 `LambdaUpdateWrapper` 执行 SQL 原生累加：`balance = balance + initialCoins`，防止并发覆盖问题。
*   **账务流水**：同步向 `tb_coin_log` 插入一条类型为 `4` (系统调整) 的记录，便于审计。

---

## 2. 老人每日登录签到奖励逻辑

旨在通过每日微量激励提升老人用户的日活（DAU），并采用 Redis 机制防止接口重复调用。

### 2.1 触发机制
*   **前端入口**：老人登录后，通过弹出的“奖励领取”弹窗或手动点击“签到”按钮触发。
*   **API 接口**：`POST /user/daily-login-reward`

### 2.2 技术方案
*   **代码位置**：`ClientUserServiceImpl.dailyLoginReward()`
*   **防重复领取（Redis 定时锁）**：
    *   **键值设计**：使用 `daily_login:yyyy-MM-dd:userId` 作为唯一标识。
    *   **时效逻辑**：签到成功后，该键在 Redis 中的 TTL（过期时间）自动设置为**截止今日 23:59:59 的剩余时长**。
    *   **效果**：用户在同一自然日内多次请求接口，将被 Redis 拦截并抛出“今日已签到”业务异常。
*   **角色保护**：接口内部包含 `user.getRole().equals("老人")` 的硬逻辑硬校验，越权请求将返回 `403`。
*   **奖励额度**：通过 `SettingsService` 获取配置参数 `daily_sign_in_reward`（系统默认值：**5**）。
*   **资产下发**：执行余额累加，并生成类型为 `4` (系统调整) 的资金流水记录。

## 3. 汇总表

| 奖励项 | 触发点 | 默认金额 | 后台配置键 | 防重策略 |
| :--- | :--- | :--- | :--- | :--- |
| **注册大礼包** | 实名认证审核通过 | 100 积分 | `elder_initial_coins` | 状态机控制（仅限 PENDING 态） |
| **每日登录** | 用户手动签到请求 | 5 积分 | `daily_sign_in_reward` | Redis 日级唯一键锁 |

# 亲情绑定接口文档

## 1. 业务概述
亲情绑定功能允许子女（代理人）发起与老人（父母）的绑定申请，由管理员在后台进行审核。审核通过后，子女可以代理老人进行操作。



---

## 2. 接口定义

### 2.1 获取亲情绑定申请列表

**接口描述**: 分页获取亲情绑定申请记录，支持按状态筛选。

**请求方式**: `GET /api/user-relations`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | number | 是 | 页码，默认1 |
| pageSize | number | 是 | 每页数量，默认10 |
| status | number | 否 | 状态筛选 (0:待审核, 1:已通过, 2:已拒绝) |
| keyword | string | 否 | 搜索关键字（申请人/目标人姓名或手机号） |

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "total": 100,
    "list": [
      {
        "id": 1,
        "childId": 1001,
        "childName": "张三",
        "childPhone": "13800138000",
        "parentId": 2001,
        "parentName": "张老头",
        "parentPhone": "13900139000",
        "proofImg": "https://oss.../proof1.jpg?Signature=...,https://oss.../proof2.jpg?Signature=...", // 证明图片URL，已签名的多图，逗号分隔
        "status": 0,
        "createTime": "2024-12-25 10:00:00"
      }
    ]
  },
  "msg": "success"
}
```

### 2.2 审核亲情绑定申请

**接口描述**: 对待审核的申请进行通过或拒绝操作。

**请求方式**: `POST /api/user-relations/{id}/audit`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| status | number | 是 | 审核动作 (1:通过/向老人发起确认, 2:拒绝) |
| rejectReason | string | 否 | 拒绝原因 (当 status=2 时必填) |

**响应示例**:
```json
{
  "code": 200,
  "data": null,
  "msg": "操作成功"
}
```

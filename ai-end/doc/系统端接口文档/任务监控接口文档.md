# 任务监控接口文档

## 1. 概述
任务监控模块用于管理员查看系统中的所有任务，并进行管理操作（如强制关闭）。

## 2. 数据库映射与分析

根据 `pages/TaskMonitor.tsx` 的需求和 `doc/数据库现有表详情.md` 的现有结构，分析如下：

### 核心表: `tb_task`
| 前端字段 | 数据库字段 | 说明 | 现状/建议 |
|Keys|Values|Remarks|Status/Suggestion|
|---|---|---|---|
| id | id | 任务ID | 现有 |
| title | title | 标题 | 现有 |
| type | type | 任务类型 (陪聊/保洁/跑腿/医疗陪护/其他) | **现有** - varchar(20) |
| coins | price | 悬赏金额 | 现有 |
| creatorId | publisher_id | 发布者ID | 现有 |
| volunteerId | volunteer_id | 志愿者ID | 现有 |
| status | status | 状态 | 现有 (0:待接取 1:进行中 2:待确认 3:已完成 4:申诉中 5:已取消) |
| cancelReason | cancel_reason | 取消/关闭原因 | **现有** - varchar(255)，用于存储强制关闭时的原因 |
| publishTime | create_time | 发布时间 | 现有 |
| deadline | deadline | 截止时间 | **现有** - datetime |

### 关联查询
- **发布者信息**: 关联 `sys_user` 通过 `publisher_id` 获取 `realName`, `phone`, `nickname`。
- **志愿者信息**: 关联 `sys_user` 通过 `volunteer_id` 获取 `realName`, `nickname`。

## 3. 接口定义

### 3.1 获取任务列表

**URL**: `/mission/list`  
**Method**: `GET`  
**Description**: 分页获取任务列表，支持多维度筛选。

**请求参数 (Query Params)**:

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| page | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页条数，默认 10 |
| keyword | string | 否 | 搜索关键词 (任务ID / 标题 / 发布人手机) |
| status | string | 否 | 状态筛选 (ALL, PENDING, IN_PROGRESS, etc.) |
| type | string | 否 | 类型筛选 (ALL, CHAT, CLEANING, etc.) |
| date | string | 否 | 发布日期 (YYYY-MM-DD) |

**响应数据**:

```typescript
{
  "code": 200,
  "msg": "success",
  "data": {
    "total": 100, // 总条数
    "list": [
      {
        "id": "1001",
        "title": "陪老人去医院",
        "type": "MEDICAL", // 如果数据库缺失，暂返回默认或根据标题推断
        "status": "PENDING",
        "coins": 50,
        "publishTime": "2024-05-20 10:00:00",
        "deadline": "2024-05-21 18:00:00",
        "creatorName": "张三", // 昵称
        "creatorRealName": "张三丰", // 真实姓名
        "creatorPhone": "13800138000",
        "volunteerName": "李四", // 接单人 (可选)
        "volunteerPhone": "13900139000" // (可选)
      }
      // ...
    ]
  }
}
```

### 3.2 强制关闭任务

**URL**: `/mission/force-close`  
**Method**: `POST`  
**Description**: 管理员强制关闭某个进行中或待接单的任务，资金退回。

**请求参数 (Body)**:

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| taskId | string | 是 | 任务ID |
| reason | string | 是 | 关闭原因 |

**响应数据**:

```json
{
  "code": 200,
  "msg": "操作成功"
}
```

### 3.3 获取任务详情

**URL**: `/mission/{id}`
**Method**: `GET`
**Description**: 获取单个任务的详细信息，包括日志、相关人员信息等。

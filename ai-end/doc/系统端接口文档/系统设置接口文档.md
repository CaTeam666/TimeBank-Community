# 系统设置接口文档

## 1. 接口概述

Base URL: `/api`

本模块负责获取和更新系统的核心参数配置。配置项以键值对 (Key-Value) 形式存储。

## 2. 数据结构

### SystemSettings (系统设置对象)

前端交互时通常使用如下 JSON 对象结构：

```typescript
interface SystemSettings {
  elderInitialCoins: number;      // 老人初始赠送时间币
  dailySignInReward: number;      // 每日签到基础奖励
  monthlyRank1Reward: number;     // 月度排行榜第一名奖金
  monthlyRank2Reward: number;     // 月度排行榜第二名奖金
  monthlyRank3Reward: number;     // 月度排行榜第三名奖金
  monthlyRank4Reward: number;     // 月度排行榜第四名奖金
  monthlyRank5Reward: number;     // 月度排行榜第五名奖金
  transactionFeePercent: number;  // 任务发布预扣手续费比例 (%)
  zombieTaskTimeoutHours: number; // 僵尸任务超时判定时长（小时）—— 任务发布后超过此时长仍未被接单，系统将自动关闭任务并将冻结的时间币退还给发布者
  taskAutoAcceptDays: number;     // 任务完成自动验收期限（天）—— 志愿者提交凭证后，若老人在此期限内未确认验收，系统将自动验收并发放时间币给志愿者
  familyBindingMaxLimit: number;  // 老人亲属绑定数量上限（人）—— 每位老人最多可被绑定的子女亲属数量，超过此限制新的绑定申请将被拒绝
  proxyDailyActionLimit: number;  // 代理人每日操作次数限制（次）—— 子女代理老人操作时，每天最多可执行的操作次数，超出后当天不可再操作
  orderCancelTimeoutHours: number;// 订单未核销自动取消时长（小时）—— 商品兑换订单生成后，若在此时长内未完成核销，系统将自动取消订单并将冻结的时间币退回用户可用余额
}
```

### 数据库存储映射 (sys_config Key 映射)

后端数据库 `sys_config` 表中的 `config_key` 对应关系：

| 前端字段 | 数据库 config_key | 说明 | 数据类型 |
| --- | --- | --- | --- |
| `elderInitialCoins` | `elder_initial_coins` | 老人初始赠送时间币 | int |
| `dailySignInReward` | `daily_sign_in_reward` | 每日签到基础奖励 (自动被发放到老人当日登入签到中) | int |
| `monthlyRank1Reward` | `monthly_rank_1_reward` | 月度排行榜第一名奖金 | int |
| `monthlyRank2Reward` | `monthly_rank_2_reward` | 月度排行榜第二名奖金 | int |
| `monthlyRank3Reward` | `monthly_rank_3_reward` | 月度排行榜第三名奖金 | int |
| `monthlyRank4Reward` | `monthly_rank_4_reward` | 月度排行榜第四名奖金 | int |
| `monthlyRank5Reward` | `monthly_rank_5_reward` | 月度排行榜第五名奖金 | int |
| `transactionFeePercent` | `transaction_fee_percent` | 任务发布预扣手续费比例 | int |
| `zombieTaskTimeoutHours` | `zombie_task_timeout_hours` | 僵尸任务超时判定时长（小时）—— 任务发布超过此时长未接单则自动关闭退款 | int |
| `taskAutoAcceptDays` | `task_auto_accept_days` | 任务完成自动验收期限（天）—— 老人未确认则系统自动验收发放积分 | int |
| `familyBindingMaxLimit` | `family_binding_max_limit` | 老人亲属绑定数量上限（人）—— 超过后新绑定申请将被拒绝 | int |
| `proxyDailyActionLimit` | `proxy_daily_action_limit` | 代理人每日操作次数限制（次）—— 子女代理操作超出后当天将被限制 | int |
| `orderCancelTimeoutHours` | `order_cancel_timeout_hours` | 订单未核销自动取消时长（小时）—— 超时未核销则自动取消并退回积分 | int |

## 3. 接口详情

### 3.1 获取所有系统配置

获取当前生效的所有系统配置参数。

- **URL**: `/settings`
- **Method**: `GET`
- **Auth**: Required (Admin)

#### Response

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "elderInitialCoins": 100,
    "dailySignInReward": 5,
    "monthlyRank1Reward": 500,
    "monthlyRank2Reward": 300,
    "monthlyRank3Reward": 100,
    "monthlyRank4Reward": 50,
    "monthlyRank5Reward": 30,
    "transactionFeePercent": 0,
    "zombieTaskTimeoutHours": 24,
    "taskAutoAcceptDays": 3,
    "familyBindingMaxLimit": 3,
    "proxyDailyActionLimit": 5,
    "orderCancelTimeoutHours": 24
  }
}
```

### 3.2 更新系统配置

批量更新系统配置参数。仅需发送需要修改的字段，或者发送完整对象。

- **URL**: `/settings`
- **Method**: `POST`
- **Content-Type**: `application/json`
- **Auth**: Required (Admin)

#### Request Body
请求参数为一个包装对象，包含管理员密码和要更新的配置对象：

```json
{
  "password": "admin_password_here",
  "settings": {
    "elderInitialCoins": 100,
    "dailySignInReward": 10,
    "monthlyRank1Reward": 1000,
    "monthlyRank2Reward": 800,
    "monthlyRank3Reward": 500,
    "monthlyRank4Reward": 300,
    "monthlyRank5Reward": 100,
    "transactionFeePercent": 5,
    "zombieTaskTimeoutHours": 24,
    "taskAutoAcceptDays": 3,
    "familyBindingMaxLimit": 3,
    "proxyDailyActionLimit": 5,
    "orderCancelTimeoutHours": 24
  }
}
```

#### Response

```json
{
  "code": 200,
  "message": "系统配置已保存生效",
  "data": null
}
```
